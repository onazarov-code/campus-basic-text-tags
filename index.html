Module 1: Introduction to Web Programming & Version Control

1.1 Web history

Internet
The internet began as four networked computers in 1969. Today, the internet is the largest and most popular computer network, spanning the entire globe.

FTP / File Transfer Protocol
FTP (File Transfer Protocol) was an early way to transfer files over the internet.

World Wide Web / web
In the early 1990s Tim Berners-Lee (at CERN) developed the World Wide Web — a convenient way for computers to communicate and link documents. A webpage is a document viewed in a web browser; related webpages form a website. A web server is a program that serves webpages to browsers. HTML (HyperText Markup Language) is the standard markup language for web documents. Hypertext contains links to other text, images, or media; markup adds structure and metadata for formatting and links. :contentReference[oaicite:1]{index=1}

Web browser, W3C, WHATWG
A web browser downloads HTML from a web server and displays it. The World Wide Web Consortium (W3C) and the WHATWG are organizations that develop and maintain web standards; WHATWG now produces the HTML Living Standard. :contentReference[oaicite:2]{index=2}

Notes: figures and animations in the zyBooks chapter illustrate early web pages, the web’s "spider-web" interconnections, and browser timelines.

1.2 IP addresses, domain names, and URLs

Packet & IP address
An internet packet contains source/destination IP addresses plus data. An IP address uniquely identifies a device on the internet (e.g., 198.51.100.7). IPv4 uses 32-bit addresses; IPv6 uses 128-bit addresses. :contentReference[oaicite:3]{index=3}

Domain name & DNS
A domain name (like wikipedia.org) maps to an IP address via DNS. Root DNS servers are authoritative entry points; registrars sell domain names. Top-level domains (TLDs) include .com, .net, .org and country codes like .uk, .ru. :contentReference[oaicite:4]{index=4}

URL parts
A URL (Uniform Resource Locator) includes a scheme (http:// or https://), hostname (domain), path (folders/files), optional query string (after ?), and optional fragment (after #). If a server is reached but the resource isn't found, the server returns a 404 status code (page not found). Linkrot is when previously valid links return 404. :contentReference[oaicite:5]{index=5}

1.3 HTTP

HTTP & related protocols
HTTP (HyperText Transfer Protocol) runs over TCP/IP and governs browser–server communication. HTTP/1.1 was long-standing; HTTP/2 improves speed and many sites use it; HTTP/3 (using UDP) is the next evolution. A browser performs a DNS lookup, sends an HTTP request, and receives an HTTP response which often contains the requested resource. :contentReference[oaicite:6]{index=6}

Headers, status codes, caching
HTTP requests and responses include start lines, header fields, and (optionally) message bodies. Status codes (e.g., 200, 301, 404) indicate result. Caching uses headers like Expires, Cache-Control, and ETags; DevTools can inspect requests/responses. HTTPS secures HTTP with TLS and digital certificates from certificate authorities. :contentReference[oaicite:7]{index=7}

1.4 Web trends

Important trends include mobile devices, the Internet of Things (IoT), web accessibility (making sites usable for people with disabilities), cognitive computing (AI), and the design principle of separation of concerns (structure, presentation, interaction). :contentReference[oaicite:8]{index=8}

Module 2: Introduction to HTML

2.1 HTML document structure

HTML (HyperText Markup Language)
HTML is a textual language for creating webpages. An element is an HTML structure; tags mark start/end. The minimum document includes <!DOCTYPE html>, an <html> element, a <head> (meta, title), and a <body> with renderable content. Use UTF-8 encoding and validate markup to avoid quirks mode. :contentReference[oaicite:10]{index=10}

Selected topic notes (2.2–2.8)

Meta tags
Meta elements provide metadata (charset, author, viewport) and should appear in the head in a recommended order.

<script> and <style>
Use <script> to include executable JS (or reference via src), and <style> for embedded CSS. Prefer external files for separation of concerns.

Comments & developer guidelines
HTML comments are not shown in browsers. Follow developer guidelines: close tags properly, use lowercase, quote attribute values, indent nested elements, and separate structure from presentation. Use validators to check correctness. :contentReference[oaicite:11]{index=11}

Containers: block vs inline
A container is any element with opening and closing tags. Block elements fill parent width and can contain blocks and inline elements; inline elements take only needed space and typically contain text or other inline elements. Understanding hierarchy (parent/child) is important for layout. :contentReference[oaicite:12]{index=12}
